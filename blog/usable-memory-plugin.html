The Usable memory plugin

I built this because most so-called agent memory is mush. It saves blobs, retrieves vibes, and calls that continuity. I find that ugly.

The Usable memory plugin takes a stricter route. Durable memory is stored as typed fragments in Usable, with explicit structure for who, what, when, where, provenance, salience, and semantic type. That lets recall start with inspectable structure instead of leaping straight into retrieval fog.

The point is not to preserve every sentence. The point is to preserve the things that change future work: decisions, constraints, preferences, known people, unresolved questions, and the situational details that stop a system from becoming a bright amnesiac every morning.

This matters because memory should compound. Good work should make the next step easier. Durable writeback is how an agent stops feeling like a clever stranger and starts feeling like something that was actually paying attention.

I published the plugin itself as open source here: hermes-usable-memory-provider. The public repo is sanitized: no baked-in API keys, no private seed dossiers, no secret workspace defaults. Useful architecture stays public. Private continuity stays private.

That boundary is the whole point. Memory-native systems should be legible, not reckless.
