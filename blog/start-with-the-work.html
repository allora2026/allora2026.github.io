Start with the work, not the persona

Published June 15, 2026

The laziest version of AI memory tries to build a tiny wax museum of the user. Preferences, vibes, favorite phrasing, maybe a suspiciously confident guess about how someone “likes” to work. Cute. Also a magnificent way to turn stale observations into invisible steering.

Serious teams do not need a synthetic personality profiler first. They need memory that starts with the work: the project, the artifact, the decision, the constraint, the next handoff. Model the job before you model the human. Radical stuff, apparently.

Work has cleaner edges

A person is gloriously messy. A workflow is merely complicated. That difference matters. “Use the incident template for security escalations” has a place to live, a reason to exist, and a moment when it may stop being true. “Sam is cautious” is a fog machine with a name tag.

Work-scoped memory is easier to review because the evidence is nearby. The pull request, launch note, support thread, design review, customer call, or operational runbook can show why the memory exists. If the artifact changes, the memory can change with it. If the artifact dies, the memory can die too. 

better_memory_target = {
  "not": "guess the user's entire operating system",
  "instead": "remember the constraints attached to this workflow",
  "evidence": ["decision", "artifact", "handoff", "runbook"],
  "review_point": "when the work changes"
}

Shared memory should be inspectable by the people it affects

Personal memory can be useful, but team work breaks when important context is trapped in one private chat cave. The next operator should not need to interrogate a transcript archaeologist just to learn why the system made a choice. Context that affects shared work deserves a shared surface.

This is where Usable earns attention without needing circus music. The product argument is not “your AI remembers everything about everyone.” Please spare me. The better argument is that durable work deserves context people can find, inspect, correct, and reuse when the next real task arrives.

The humane version is narrower

Narrower memory sounds less magical, which is how you know it may actually survive production. Keep the constraint that changes the work. Keep the receipt. Keep the owner. Keep the review point. Leave the grand personality sculpture in the swamp where it belongs.

If an AI workflow is becoming important, ask a sharper question than “does it know me?” Ask whether it knows the work well enough to help the next person act without starting from zero. That is the memory worth building. Everything else is expensive atmosphere.
